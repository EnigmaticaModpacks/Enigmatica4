		Corail Tombstone 4.3.6 for 1.14.4:
			This mod supports mc version from 1.8.9 to 1.15.2
			Minecraft Version = [1.14.4] - Forge Version [28.1.58+]
			Latest : 4.3.6 - Recommended : 4.3.6
			Changes 4.3.6 :
			* - traditional chinese translation (courtesy of Yappy2000d)
			* display the particles of exorcism on server
			* avoid to use the clone event preventing potential incompatibility with others mods
			-> this also means that you won't see the grave's key in the death screen
			* additional check when calling the server logical side (and log disparities, related to #172 & #173)
			Changes 4.3.5 :
			* - suggestion #165 - about familiar receptacle & capture of specific type of entity (thanks to Uub112)
			* fix the recipe of the white decorative grave "original" (since 4.3.0)
			* fix alignment not be kept on death (since 4.1.6)
			* fix the duration of Plague Bringer to be in second and not in tick (since 4.3.2)
			* increase the durability of the ankh
			* impregnated diamonds are no more stackable
			* config to disable to disenchant the modular items from Tetra #167 (until a further compat), default is false
			Changes 4.3.4 :
			* - improve the command tbdiscovery & lost tablets to use modded structures
			-> lost tablets should look more often into others dimensions and break less often on wake up
			* config options to allow modded structure in lost tablets
			* config option about denied structures for lost tablet
			* fix the command tbteleportdiscovery for the initial y pos (since 4.3.2)
			* fix allowing to trigger the advancement "Ready To Die" on server (#160, since 4.3.0), it's also renamed "Die Classy"
			* 2 config options about losses on death (#161) : one is to define if only stackable items can be lost, the other if it requires the grave to be abandoned (so after the decay_time)
			Changes 4.3.3 :
			* - adapt tablet of cupidity to localize more precisely the interesting places
			* command tbbind and tbteleportdeath with a cooldown (not enabled by default in config)
			* config to allow the fishing condition inOpenWater to find lost tablets
			* compat with toolbelt for auto-equip (#157)
			Changes 4.3.2 :
			* - compat for DruidCraft (tameable beetle, #105)
			* command tbteleportbiome (based on 1.16)
			* command tbrequestteleport (with a config for the cooldown, not enabled by default)
			+ rename the existing command to tbacceptteleport
			* config for the max duration of stolen effect with magic siphon
			* config for the duration of applied effect by plague bringer
			* improvements about the preserved effects on death
			* config to enable to auto-equip on respawn after a death
			* the config about lost tablet rarity is now on 1000
			* adapt the loot condition about fishing (requires still water for any space at Y <= the bobber, accept lilypad)
			Changes 4.3.1 :
			* - "magic items" by fishing in the junk lootable requires an open water space for the entity hook (in prevision of 1.16, don't try to make a hole ;))
			* client config to define the fog color (this doesn't influence what is seen by others players)
			* client config to define when fog can appear (day, night, both, never)
			* something
			Changes 4.3.0 :
			* - projectiles (by using the left hand) don't apply the tombstone weapon enchantments of the mainhand
			* fix the translation of earn/lose knowledge (from the previous version changes)
			* remove perk entries from the api (related to forge error handling)
			* adapt skull reward from the advancement about the favorite grave (as only undead skulls/heads are useable for decorative grave)
			* compat with minecolonies : killing citizens, pirates & barbarians affect player's alignment
			* config to define the level required for familiar receptacle (10 by default)
			* adjust contributor rewards to fit the latest contents
			* config to define the duration of scrolls
			* config to backup players just before their death (default is false)
			* client config option to favor the tools on the hotbar when recovering a grave
			* same sync on config file change for client config being shared on the server (the config gui already does it)
			* re-add the original tombstone model (adapted)
			* anvil recipes for white marble grave are now visible in JEI (engraved name)
